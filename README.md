This project was for my CS 478 course during my Spring 2026 semester at the University of Illinois Chicago. I independently designed and developed two Android applications, FunCenter and FunClient, that communicate through a bound service using Android Interface Definition Language (AIDL). The project simulates a media-sharing system where one application provides images and audio clips while the second application connects to the service to retrieve images and control media playback remotely.

The project demonstrates Android development skills including interprocess communication (IPC), bound services, AIDL interfaces, MediaPlayer integration, lifecycle management, and thread-safe service design. I implemented both the service-side and client-side functionality, including image retrieval by index, audio playback controls (play, pause, resume, and stop), service binding and unbinding, and concurrent handling of image requests while audio playback remained active.

The FunCenter application stores multiple pictures and audio clips locally and exposes an API through a custom bound service. FunClient binds to the service using a ServiceConnection object and provides a user interface for interacting with the service in real time. As required by the course, Claude AI was used as a development assistant to support debugging and guide implementation choices. It was used as a learning tool to improve development efficiency while maintaining a strong understanding of all implemented code.

The complete project is available in both the FunClient.zip and FunCenter.zip files, with additional source files separated for clarity and easier navigation through the core implementation.
